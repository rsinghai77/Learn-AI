## 5. Data Confidentiality and Regulatory Compliance

> This is one of the most important and underappreciated skill areas for finance professionals using AI tools. Getting this wrong can end careers and create legal liability.

### The Regulatory Landscape in 2026

**SEC 2026 Examination Priorities — What They Are Actively Looking For:**
- Whether AI-driven recommendations align with regulatory expectations and client interests
- **"AI-washing" enforcement:** Firms claiming AI capabilities must actually use them as described
- Whether firms can explain AI-driven decisions to examiners with full audit trails

**FINRA 2026 Oversight — Key Rules That Apply to AI:**
- **Rule 3110 (Supervision):** Your AI tools must be part of a "reasonably designed supervisory system" — firms must oversee AI model integrity, reliability, and accuracy
- **Rule 3120 (Supervisory Control):** When AI moves from content generation to autonomous action, oversight obligations shift materially
- **Key risks flagged:** Hallucinations in regulatory interpretation, bias from outdated training data, autonomous agents acting beyond intended scope

### The VALID Framework — Memorize This

Every finance professional using AI tools in 2026 should internalize this:

```
V — Validate all outputs before distributing to clients or stakeholders
A — Avoid personal information in unapproved AI systems
L — Look for hallucinations and bias in every output
I — Insulate sensitive data from external AI tools
D — Disclose AI usage when appropriate (clients, regulators)
```

### Data Classification: What Goes Where

| Data Type | Risk Level | Rule |
|---|---|---|
| Client PII (names, SSNs, account numbers) | High | NEVER input into unapproved external AI tools |
| Proprietary trading strategies | High | Internal-only AI systems with data walls |
| Client portfolio holdings | High | Requires approved "walled garden" environments |
| Aggregated/anonymized market data | Low-Medium | Can often be used with standard AI tools |
| Publicly available company data | Low | Generally safe for external AI tools |

### Critical Compliance Skills to Develop

1. **Understanding KYC/AML requirements** — Know what AI-assisted customer verification systems must demonstrate to regulators
2. **Audit trail design** — How to structure AI workflows so every decision is logged and explainable
3. **AI risk classification** — Ability to classify use cases as High/Moderate/Low risk
4. **Vendor due diligence** — How to evaluate an AI vendor's SOC 2 compliance, data handling policies, and privacy posture
5. **GDPR/CCPA basics** — Personal data rights and how they constrain AI systems
6. **Explainability requirements** — For credit decisions, investment recommendations, and other regulated outputs, AI must produce traceable reasoning

### Practical Rule for a Finance Student Starting Out

> **Rule of thumb:** If you would not be comfortable reading a data input aloud in a compliance meeting, do not put it into an AI tool — unless you are using a firm-approved, air-gapped system with confirmed data protections.

## 6. How to Use Claude Code Without Being a Developer

### The Most Important Insight

Claude Code is fundamentally different from traditional software development tools. You do not write code — you describe what you need in plain English, and Claude writes, runs, and manages the code for you. Your value is your **domain knowledge, not your syntax.**

Think of it like this:
```
Traditional software development:
  You → [learn code] → write code → debug → deploy

Claude Code workflow:
  You → [describe what you need in finance terms] → Claude builds it → You verify outputs
```

### The Mental Model That Works

Think of Claude Code as a brilliant but inexperienced junior analyst who:
- Has read every programming textbook ever written
- Has never worked in a finance firm
- Will do **exactly** what you describe, literally
- Needs your domain expertise to build something that actually works correctly
- Will sometimes make plausible-looking mistakes that only your finance knowledge can catch

### Practical Workflow for a Finance Professional

**Step 1 — Define the task in business terms**
```
Wrong: "Write Python code to calculate DCF"
Right: "Build a DCF valuation model for a manufacturing company.
        Use a 5-year explicit forecast period, terminal value using
        Gordon Growth Model with a 2.5% perpetuity growth rate,
        WACC inputs I can adjust, and output a sensitivity table
        showing NPV across different WACC and growth rate combinations."
```

**Step 2 — Provide your domain constraints explicitly**
```
"The model must follow these rules:
- Use unlevered free cash flow (EBIT × (1-tax rate) + D&A - CapEx - ΔNWC)
- Show the bridge from enterprise value to equity value
- Flag if EBITDA margin assumptions exceed industry benchmarks by >500bps"
```

**Step 3 — Review outputs against your financial knowledge**
This is where your Culverhouse education pays off. Ask yourself:
- Does the balance sheet balance?
- Do the cash flows make economic sense?
- Are the valuation multiples in a reasonable range?
- Would a CFO or analyst question any of these numbers?

**Step 4 — Iterate with specific feedback**
```
"The EBITDA margin in Year 3 is 42% — that's implausible for this
industry. Cap it at 28% and recalculate downstream numbers."
```

### Getting Started Without Overwhelm: The 3-Day Method

**Day 1:** Use Claude Code for one existing task you do anyway.
- Example: "I have this Excel export of monthly expenses in a CSV file. Create a summary by category and flag any items over $5,000."
- No new skill needed. Just describe what you want.

**Day 2:** Ask Claude to explain what it built.
- "Explain each step of the code you wrote in plain English."
- Build your mental model without pressure.

**Day 3:** Modify one thing.
- "Change the threshold from $5,000 to $10,000."
- Learn that changing code is just changing instructions.

| Task | Traditional Time | With Claude Code |
|---|---|---|
| Build a variance analysis template | 3–4 hours in Excel | 10-minute description |
| Write a script to pull stock prices into a spreadsheet | Requires developer | 5-minute description |
| Analyze 50 earnings call transcripts | Days | Hours with AI pipeline |

## 7. Building Finance Applications with AI: What's Possible

### AI Agents You Can Build for Finance

AI agents are software systems that take actions autonomously on your behalf — think of a very capable assistant that can work through multi-step processes without you supervising every step.

**Realistic applications you can build with Claude Code as a Finance grad:**

| Application | What It Does | Complexity |
|---|---|---|
| **Portfolio Monitor Agent** | Watches a set of stocks, alerts you when price moves, news breaks, or a valuation metric triggers | Low |
| **Earnings Analysis Agent** | When earnings drop, pulls the transcript, extracts key metrics, compares to estimates, and drafts an analysis | Medium |
| **Budget Variance Detector** | Compares actual vs. budget monthly, generates narratives explaining each variance | Low |
| **Due Diligence Agent** | Given a company name, pulls financials, news, SEC filings, and generates a structured summary | Medium |
| **FX Risk Monitor** | Monitors currency exposure and alerts when hedge ratios drift beyond thresholds | Medium |
| **Compliance Document Checker** | Scans reports and communications for regulatory risk keywords and flags them | Low |

### MCP Servers: Connecting AI to Financial Data

MCP (Model Context Protocol) servers are connectors that give AI tools like Claude direct access to live data and systems. For finance, this means:

**Free/Open Source MCP Servers Available Now:**
- **financial-datasets MCP** (GitHub): Real-time stock prices, income statements, balance sheets, cash flows
- **Yahoo Finance MCP** (GitHub): Historical prices, company info, options data, market news
- **Alpha Vantage MCP**: Real-time and historical market data
- **FRED API integration**: Macroeconomic data (GDP, inflation, interest rates) from the Federal Reserve

**Enterprise MCP Servers:**
- **Drivetrain MCP**: Live FP&A data, budget variance detection, board pack generation
- **FactSet MCP**: Institutional-grade financial data (used by major investment banks)
- **LSEG MCP**: London Stock Exchange Group data
- **Microsoft Dynamics 365 Finance MCP**: ERP integration with security-scoped access

**What this means practically:** You can have a conversation with Claude like:
```
"Pull Apple's last 4 quarters of financial data, calculate free cash flow
growth, and tell me if the current P/FCF multiple is above or below its
5-year historical average."
```
...and get a real-time, data-backed answer in seconds — without building anything.

### The Tools Stack for a Finance + AI Developer

```
Data Layer:     MCP Servers (Yahoo Finance, Alpha Vantage, FRED, FactSet)
                    ↓
AI Layer:       Claude (via Bedrock or API) + MCP Clients
                    ↓
Application:    Web dashboard / Excel / Python notebook / Spreadsheet
                    ↓
Output:         Reports, alerts, models, recommendations
```

You do not need to build the data layer (MCP servers exist). You do not need to build the AI layer (Claude exists). Your job is to design the application layer — what questions to ask, what outputs to generate, what actions to take — and that is pure finance domain work.

## 8. Finance Skills You Should Deepen

These are the finance skills that become *more* valuable, not less, when paired with AI tools. Prioritize these.

### Tier 1 — Non-Negotiable Core (Deepen These First)

**1. Financial Statement Analysis**
- Reading and interpreting income statements, balance sheets, and cash flow statements
- Understanding accounting mechanics (revenue recognition, working capital, D&A treatment)
- Why: AI models built on wrong accounting assumptions produce confidently wrong results. You are the validator.
- Resource: Invest time in CFA Level 1 accounting material even if not pursuing the charter

**2. Valuation Frameworks**
- DCF (Discounted Cash Flow): WACC calculation, terminal value, free cash flow definition
- Comparable Company Analysis (Comps): selecting peers, spreading multiples
- Precedent Transactions: M&A deal analysis
- Why: When directing AI to build valuation models, you must specify the framework precisely or the model will make arbitrary assumptions

**3. Financial Modeling Judgment**
- Understanding the structure of integrated 3-statement models
- Knowing when a model output is implausible (key quality-control skill)
- Scenario analysis and sensitivity table design
- Why: AI modeling tools (Claude, Shortcut) still hallucinate ~40% of complex model details. Your judgment is the safety layer.

**4. Risk Fundamentals**
- Market risk, credit risk, liquidity risk, operational risk
- Basic options and derivatives (hedging concepts, not full pricing theory)
- Why: Risk management is the highest-growth AI use case in financial services; your risk vocabulary directs AI effectively

### Tier 2 — High Value, Worth Developing

**5. Corporate Finance Mechanics**
- Capital structure decisions, cost of capital, M&A rationale
- Leveraged buyout (LBO) basic structure
- Why: Investment banking and private equity AI tools require you to specify these correctly

**6. Fixed Income Basics**
- Bond pricing, duration, yield curve interpretation
- Why: Critical for treasury, risk, and macro-focused AI applications

**7. Regulatory and Compliance Framework**
- How SEC, FINRA, and CFTC oversight works
- KYC/AML process and requirements
- Fiduciary standards in wealth management
- Why: RegTech is a major growth sector; compliance knowledge makes you immediately valuable in AI governance roles

**8. FP&A (Financial Planning & Analysis)**
- Budgeting, forecasting, variance analysis
- Management reporting and KPI design
- Why: FP&A automation is the fastest-growing AI use case inside corporations

## 9. Software Skills Worth Learning (Focused, Practical List)

> **Philosophy:** You do not need to become a software developer. You need enough technical fluency to direct AI tools effectively and verify their outputs. Every skill listed here has direct finance application and can be learned at the surface level needed without deep programming expertise.

### Must Learn (Investment of 4–6 weeks each)

**1. Python for Finance (Beginner Level)**

What you need:
- Variables, functions, loops, conditionals — the basics
- `pandas`: loading and manipulating financial data (DataFrames, groupby, merge)
- `numpy`: numerical calculations (returns, statistics)
- Jupyter notebooks: interactive analysis environment

What you do NOT need yet:
- Object-oriented programming
- Building production-grade systems
- Machine learning from scratch

Why: 51% of finance AI hiring managers require Python. More importantly, when Claude Code writes Python, you can read it, spot errors, and iterate effectively.

**2. SQL (Basic Queries)**

What you need:
- SELECT, WHERE, GROUP BY, JOIN — that's 90% of what you'll use
- Understanding of database tables and relationships

Why: Financial data lives in databases. SQL lets you extract it — or direct Claude Code to do so precisely.

**3. Excel (Advanced Features)**

What you need (beyond what you likely have):
- Power Query for data transformation
- INDEX/MATCH, XLOOKUP (faster than VLOOKUP)
- Basic VBA (enough to understand what Claude generates)

Why: Excel remains the modeling backbone of finance. AI-generated Excel models need someone who knows Excel deeply to audit them.

### Strongly Recommended (2–3 weeks each)

**4. Power BI or Tableau (Choose One)**
- Connecting to data sources, building dashboards, using AI-assisted forecasting features
- Why: Visualization is now expected for finance analysts; AI generates the underlying data, you design the story

**5. Git / GitHub (Basics)**
- What a repository is, how to commit changes, how to share code
- Why: All Claude Code-generated projects should be version-controlled; this is the industry standard

**6. Basic Understanding of APIs**
- What an API is, what a request/response looks like (no coding required)
- Why: MCP servers are built on APIs; understanding the concept lets you use them intelligently

### Optional but High-Value

**7. Google Colab / Jupyter Notebooks**
- Cloud-based Python environment — no installation required
- Why: The easiest way to run AI-assisted financial analysis without setting up a development environment

**8. Basic JSON / YAML**
- Reading configuration files (not writing them from scratch)
- Why: MCP servers and AI agent configurations use these formats
